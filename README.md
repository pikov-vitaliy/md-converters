# Конвертеры PDF / HTML → Markdown

Комплект из двух утилит для перевода отчётов в Markdown:

| Команда | Что делает |
|---------|------------|
| `pdf2md`  | PDF → Markdown |
| `html2md` | HTML → Markdown (есть псевдоним `htmltomd`) |

Под капотом — библиотека [MarkItDown](https://github.com/microsoft/markitdown)
от Microsoft. Результат всегда кладётся **рядом с исходником**: то же имя,
расширение `.md`.

---

## Содержимое папки

```
md-converters\
├─ convert_pdf_to_md.py     — утилита PDF → Markdown
├─ convert_html_to_md.py    — утилита HTML → Markdown
├─ install.ps1              — установщик (зависимости + команды)
├─ README.md               — эта инструкция
└─ examples\
   ├─ sample-report.html   — пример входного отчёта
   └─ sample-report.md     — как он выглядит после конвертации
```

---

## Быстрый старт

1. Откройте **PowerShell** в этой папке (в Проводнике: правый клик по пустому
   месту внутри папки → «Открыть в терминале»).
2. Выполните установщик:

   ```powershell
   powershell -ExecutionPolicy Bypass -File .\install.ps1
   ```

3. Откройте **новое** окно PowerShell (чтобы команды подхватились).
4. Проверьте:

   ```powershell
   html2md .\examples\sample-report.html
   ```

Готово — рядом появится `sample-report.md`.

## Как пользоваться

### html2md — HTML в Markdown

| Команда | Что делает |
|---------|------------|
| `html2md` | спросит имя файла, папку или маску |
| `html2md отчёт.html` | один файл (расширение можно не писать: `html2md отчёт`) |
| `html2md *` | все HTML в текущей папке |
| `html2md report-*` | все, чьё имя начинается на `report-` |
| `html2md C:\reports` | все HTML в указанной папке |
| `html2md C:\reports -r` | то же, плюс все вложенные папки |
| `html2md *.html -f` | перезаписать уже существующие `.md` |
| `html2md отчёт.html --no-frontmatter` | без YAML-шапки |
| `html2md --help` | полная справка |

Можно передать сразу несколько файлов: `html2md a.html b.html c.htm`.

#### Пример вывода

```text
PS C:\reports> html2md *
Конвертирую vulnerability-report.html ...
Готово: C:\reports\vulnerability-report.md
Конвертирую compliance-2026-05-22.html ...
Готово: C:\reports\compliance-2026-05-22.md
Итого: сконвертировано 2 из 2.
```

#### Что получится (фрагмент `examples\sample-report.md`)

```markdown
---
title: "Пример отчёта об уязвимостях — demo-project"
source: "sample-report.html"
converted: 2026-06-10
generator: html2md (MarkItDown)
---

# Отчёт об уязвимостях — demo-project

## Сводка

* Всего находок: 3
* Критических: 1

## Находки

| Пакет | Уровень | Описание | Исправление |
| --- | --- | --- | --- |
| left-pad | критический | Удалённое выполнение кода | обновить до 1.3.0 |
```

### pdf2md — PDF в Markdown

Работает точно так же, как `html2md`, и понимает те же флаги — маски,
папки, рекурсию, `-f`, `--no-frontmatter`.

| Команда | Что делает |
|---------|------------|
| `pdf2md` | спросит имя файла, папку или маску |
| `pdf2md документ.pdf` | один файл (расширение можно не писать) |
| `pdf2md a.pdf b.pdf` | несколько файлов |
| `pdf2md *` | все PDF в текущей папке |
| `pdf2md C:\docs -r` | все PDF в папке и во вложенных |
| `pdf2md *.pdf -f` | перезаписать уже существующие `.md` |
| `pdf2md --help` | полная справка |

```text
PS C:\docs> pdf2md инструкция.pdf
Конвертирую инструкция.pdf ...
Готово: C:\docs\инструкция.md
```

> Единственное отличие от `html2md` — у PDF нет проблемы с кодировками,
> поэтому пункт «автоопределение кодировки» к нему не относится. Все
> остальные защиты ниже работают одинаково для обеих утилит.

## Встроенные защиты (`html2md` и `pdf2md`)

Эти механизмы добавлены специально, чтобы не было сюрпризов на больших пачках.

### 1. Защита от перезаписи

Если рядом уже есть `.md`, файл **не перезаписывается молча** — он пропускается:

```text
[пропуск] report.md уже есть (-f / --force для перезаписи)
Итого: сконвертировано 0 из 2, пропущено 2.
```

Так не потеряются правки, которые вы внесли в `.md` руками. Чтобы всё-таки
перезаписать — флаг `-f` / `--force`. В интерактивном режиме утилита
спросит один раз на всю пачку:

```text
2 файл(ов) уже имеют .md. Перезаписать? (y = да / Enter = пропустить):
```

### 2. Пропуск служебных папок при `-r`

При рекурсии (`-r`) утилита **не заходит** в служебные каталоги, где обычно
лежат сотни ненужных файлов: `node_modules`, `.git`, `.next`, `dist`, `build`,
`venv`, `.venv`, `__pycache__`, `.idea` и т. п. В репозитории не нахватаете
мусора.

### 3. Автоопределение кодировки (только `html2md`)

Если инструмент выдал HTML в `windows-1251` (или другой кодировке) без
правильного `<meta charset>`, кириллица **не превратится в «кракозябры»** —
утилита сама определит кодировку и перекодирует, пометив это в выводе:

```text
Готово: C:\reports\gost-report.md (перекодировано из cp1251)
```

### 4. YAML front-matter

В начало каждого `.md` добавляется блок с источником и датой — удобно для базы
знаний / Obsidian:

```yaml
---
title: "Заголовок из <title> отчёта"
source: "имя-исходного-файла.html"
converted: 2026-06-10
generator: html2md (MarkItDown)
---
```

Поле `generator` показывает, какой утилитой сделан файл (`html2md` или
`pdf2md`). `title` берётся из `<title>` (HTML) или метаданных PDF; если
заголовка нет — строка просто опускается. Не нужен front-matter — отключите
флагом `--no-frontmatter`.

### 5. Чистка вывода

После конвертации Markdown прибирается: убираются хвостовые пробелы,
схлопываются лишние пустые строки, обрезаются пустые края файла.

### 6. Итог и список ошибок

На пачке из нескольких файлов в конце печатается сводка, а всё, что не
удалось, перечисляется отдельно:

```text
Итого: сконвертировано 1 из 2, ошибок 1.
Не удалось сконвертировать:
  - C:\reports\битый-файл.html
```

## Перенос на другой компьютер

1. **Скопируйте всю папку** `md-converters` на новый компьютер — куда удобно
   (например, снова в `Документы`). Путь может быть любым: установщик
   сам пропишет команды на то место, куда вы положили папку.

2. **Установите Python 3.10+** (если ещё нет):
   - скачайте с <https://www.python.org/downloads/>;
   - при установке поставьте галочку **«Add python.exe to PATH»**.

   Проверка в новом окне PowerShell: `python --version`.

3. **Запустите установщик** из папки комплекта:

   ```powershell
   powershell -ExecutionPolicy Bypass -File .\install.ps1
   ```

   Он сам: найдёт Python → поставит `markitdown[pdf]` → пропишет команды
   `pdf2md` / `html2md` в ваш профиль PowerShell → проверит на примере.

4. **Откройте новое окно PowerShell** и пользуйтесь.

Установщик можно запускать повторно сколько угодно раз — старые строки он
аккуратно убирает и прописывает заново.

## Если что-то пошло не так

**`install.ps1 нельзя запустить, политика выполнения...`**
Запускайте именно так (с обходом политики только для этого файла):
```powershell
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

**`Python не найден`**
Установите Python с галочкой «Add to PATH» (см. шаг 2 выше) и запустите
установщик снова. Заглушку Python из Microsoft Store установщик игнорирует.

**Команды `pdf2md` / `html2md` не находятся**
Откройте **новое** окно PowerShell — профиль читается при запуске. Или в
текущем окне выполните `. $PROFILE`.

**Изменили расположение папки или обновили Python**
Просто запустите `install.ps1` ещё раз из новой папки — он перепропишет пути.

**Markdown с кракозябрами в консоли, но в файле всё нормально**
Это особенность отображения консоли; в самом `.md` (UTF-8) текст корректный.

## Удаление

1. Откройте профиль: `notepad $PROFILE`.
2. Удалите блок между строками
   `# >>> md-converters >>>` и `# <<< md-converters <<<`.
3. При желании удалите папку `md-converters` и саму библиотеку:
   `python -m pip uninstall markitdown`.

---

## Что нужно для работы (кратко)

- **Windows** + **PowerShell** (есть в системе по умолчанию).
- **Python 3.10+** с галочкой «Add to PATH».
- Пакет **`markitdown[pdf]`** — ставится установщиком автоматически.
