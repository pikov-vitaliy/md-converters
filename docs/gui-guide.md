# Web-GUI — справка по интерфейсу

Эта страница объясняет **каждую кнопку, галочку и поле** Web-GUI
(`tomd-gui`), чтобы не приходилось гадать. Если что-то непонятно прямо
в интерфейсе — наведите курсор на пункт: всплывёт короткая подсказка.

## Как запустить

- **Двойной клик по ярлыку «MD Converter»** на рабочем столе (создаёт
  `install.ps1`). Откроется браузер на `http://127.0.0.1:8765/`,
  чёрного окна терминала нет. Закрыли вкладку — сервер сам выключится
  примерно через минуту.
- Или из консоли в папке проекта: `tomd-gui` либо
  `python -m gui_server` (нужны GUI-зависимости: `pip install ".[gui]"`).

Сервер слушает только `127.0.0.1` (локально), проверяет заголовки
`Host` и `Origin` — сторонний сайт не может им управлять.

## Что можно подать на вход (5 способов)

| Способ | Что делает | Куда результат |
|--------|------------|----------------|
| **Перетащить / «Выбрать»** файлы в зону | Загрузка файлов в браузер, каждый → `.md` | Скачивание (кнопка ↓ у файла). Браузер не знает путь исходника, поэтому «на месте» так нельзя — для этого см. «С диска» |
| **Перетащить / выбрать архив** (`.zip`, `.7z`, `.tar`/`.tar.gz`) | Архив распаковывается на сервере, **каждый файл внутри** конвертируется в `.md` | Один `.zip` только с `.md` (кнопка «Скачать всё»), либо копия в «Папку вывода». Файлы видны по одному, каждый качается отдельно |
| **Поле URL + «Конвертировать URL»** | Тянет веб-страницу по ссылке → `.md` | Скачивание / «Папка вывода» |
| **«С диска (на месте)» → «Выбрать файлы…/папку…»** | Родной диалог ОС; путь берётся из него | `.md` пишется **рядом с исходником** (папка — рекурсивно). Ничего вводить вручную не надо |
| **«Папка → ZIP»** | Выбрать папку целиком в браузере | Один `.zip` со всеми `.md` (структура подпапок сохраняется) |

> Поддерживаемые форматы файлов: PDF, HTML/HTM, DOCX, XLSX, PPTX, CSV,
> JSON, XML, EPUB, MSG (Outlook), IPYNB (Jupyter), RSS. Плюс архивы
> `.zip`, `.7z`, `.tar`/`.tar.gz`/`.tar.bz2`/`.tar.xz` с любыми из них
> внутри. **RAR не поддерживается** (нужен внешний проприетарный
> распаковщик).

## Настройки (галочки и поля)

- **Перезаписать (force)** — перезаписывать существующий `.md`. Без
  галочки: если рядом уже лежит `.md`, файл **пропускается** (skip),
  чтобы не затереть.
- **YAML front-matter** *(по умолчанию вкл.)* — добавлять в начало
  `.md` служебную YAML-шапку: источник, дата конвертации, чем
  сконвертировано, число извлечённых PDF-таблиц и т.п. Выключите —
  получите чистый `.md` без шапки.
- **Сохранить картинки** *(по умолчанию выкл.)* — встраивать картинки
  в `.md` как `base64`. **Для PDF** извлекает встроенные растровые
  картинки (по страницам, секции «Картинки страницы N») — раньше
  PDF-картинки вообще не доставались. **Для HTML/DOCX** сохраняет уже
  встроенные. Файл становится **самодостаточным** (картинки видно во
  вьюере), **но размер сильно растёт** (одна картинка — сотни КБ —
  МБ текста). Выключено: картинок нет, плейсхолдер `![...]()`.
- **PDF-таблицы** *(по умолчанию вкл.)* — извлекать таблицы из PDF по
  геометрии (через pdfplumber) — аккуратные GFM-таблицы. Выключите —
  обычный текстовый путь MarkItDown: таблицы могут «разъезжаться».
- **URL: не проверять SSL** *(по умолчанию выкл.)* — снять проверку
  HTTPS-сертификата при конвертации URL. Нужно для внутренних хостов с
  самоподписанным сертом. ⚠️ Снимает защиту от MITM — включайте только
  для доверенных адресов.
- **Форматы** — какие типы файлов конвертировать при выборе **папки**
  (все / только PDF / HTML / DOCX / XLSX). Для одиночных файлов не
  влияет.
- **Папка вывода** — куда класть `.md`. Пусто = рядом с источником
  (для режимов «с диска» и папки). При **загрузке файлов/архива через
  браузер**: если указать путь — `.md` запишутся туда; если пусто —
  результат только скачивается (кнопкой по файлу или общим `.zip`).

## Результаты

Каждая строка результата — три области: **имя файла · кнопки · путь**.

- **Превью** — рендер Markdown (таблицы, заголовки, код). YAML-шапка
  свёрнута в блок «метаданные». Очистка от XSS (DOMPurify).
- **↓** — скачать готовый `.md`.
- **→ путь** — куда сохранён файл (для режимов «на месте» / «Папка
  вывода»). Клик по пути — копирует его в буфер обмена.
- **Скачать всё (.zip)** — общий архив со всеми `.md` (для режимов
  «Папка → ZIP» и загрузки `.zip`, когда «Папка вывода» не задана).
- Если файл не сконвертировался — строка с ❌ и текстом ошибки.

## Частые вопросы

- **Хочу сконвертировать целую папку «на месте»** → «Выбрать папку…»
  (на диске) или правый клик по папке в Проводнике → «Конвертировать
  папку в Markdown», или в консоли `tomd "C:\папка" -r`.
- **Из архива PDF получить архив Markdown** → бросьте `.zip` в зону
  загрузки → «Скачать всё (.zip)» (внутри только `.md`).
- **Картинок нет в `.md`** → включите «Сохранить картинки» (но файл
  станет большим).
