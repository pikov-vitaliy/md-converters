## Findings

### 🟡 Medium (2)

#### M-PDF-01: `sanitize_markdown` не fence-aware — искажает контент внутри ```

**Файл:** `convert_to_md.py:928–944` (`sanitize_markdown`), вызов из
`tidy:1057`
**Подтверждено экспериментально:**

```
т输入: ```\n<script>alert(1)</script>\n```
результат: ```\n\n```     ← содержимое удалено

вход: ```\n[xss](javascript:alert(1))\n```
результат: ```\n[xss]()\n```  ← URL очищен

вход: ```\ndata: application/json\n```
результат: ```\nblocked: application/json\n```  ← data: заменён
```

`sanitize_markdown` применяется ко **всему** тексту без отслеживания
fences. Для DevSecOps-базы знаний (описания уязвимостей, XSS-примеры,
эксплойт-сниппеты) это **системное искажение технического контента**.
Затронуты все форматы, где MarkItDown генерирует ``` из `<pre><code>`
(HTML, Word, PPTX), не только PDF.

**Митигация:** `--unsafe-raw-markdown` полностью отключает sanitize, но
это all-or-nothing — нет режима «санитизировать прозу, не трогать fences».

**Рекомендация:** Добавить fence-aware pass в `sanitize_markdown`:
split по ```, sanitize только нечётные сегменты (проза), чётные (code)
пропускать. Или вынести sanitize до фенсинга в PDF-пути.

**Связь:** CWE-116 (Improper Encoding); SAMM ASD2 (Defensive
Implementation).

#### M-PDF-02: Параллельные таблицы (та же Y, разная X) теряются

**Файл:** `convert_to_md.py:639,644` (`_page_blocks`)

```python
regions.sort(key=lambda r: r[0][1])   # сортировка только по top Y
if top < cursor - 1:                    # проверка перекрытия только по Y
    continue                            # вторая таблица пропускается
```

Две таблицы на одной высоте (side-by-side) — вторая пропускается, её
контент не попадает в вывод ни как таблица, ни как текст.

**Воспроизведение:** PDF с двумя таблицами на одной Y-координате (компоновка
в 2 колонки). Для типичных PDF-отчётов встречается редко (обычно таблицы
вертикально), но в широких лендскейп-документах возможно.

**Рекомендация:** При обнаружении перекрытия по Y сравнивать X-диапазоны;
если не пересекаются — обрабатывать обе. Или извлекать текст для
пропущенной таблицы через `_crop_text`.
