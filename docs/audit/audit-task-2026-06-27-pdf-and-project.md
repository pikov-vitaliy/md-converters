## 1. Что проверять

Двухуровневый аудит:

1. **Работа сессии** — PDF-конвейер, добавленный поверх `e836625`
   (коммиты `fc696b9 … 341fb03`): извлечение таблиц, псевдографика,
   чистка текста, фенсинг кода, хардинг, бамп версии.
2. **Проект целиком** — корректность, безопасность, качество, поставка.

Цель — найти регрессии, ошибки логики, дыры безопасности, неактуальную
документацию. Вердикт по каждой находке: severity + `файл:строка` +
воспроизведение + рекомендация.

## 2. Принятые ограничения / известные не-баги (НЕ репортить как новое)

Эти пункты осознанно приняты и задокументированы — проверьте обоснование,
но не заводите как новые дефекты:

- **M-01 DNS-rebinding TOCTOU** в `_check_url_allowed`/`_download_url`:
  приемлемо для локальной CLI с `--allow-private-url`. Полный фикс (пин
  IP + Host) ломает TLS-SNI. Допустимо предложить улучшение.
- **Пин `magika>=0.6.2,<0.6.3`**: 0.6.3 тянет onnxruntime без cp314-колёс.
- **Dependabot-alert по `msgpack 1.2.0`** указывает на `docs/vibe-audit/
  evidence/2026-06-13/requirements-dev-audit.txt` — это **датированный
  evidence-снимок** аудита, не живой лок. В `uv.lock` уже `msgpack 1.2.1`.
  Править evidence нельзя (исказит запись); alert — кандидат на dismiss.
- **Windows-CI без `ruff`/`uv lock --check`**: они платформонезависимы,
  покрыты Linux-джобом.
- **Cross-page row split** (строка таблицы, разорванная между страницами
  посреди ячейки, + повтор заголовка): восстанавливается не полностью
  (пример — хвост «Приложение B»). Конфликтует с разделением слипшихся
  таблиц; выбран безопасный дефолт.
- **Безбордюрные таблицы без фоновой заливки и сканы**: best-effort
  (нет геометрии). Фенсинг кода — тоже best-effort.

## 3. Точки внимания по работе сессии (PDF)

Все функции — в `convert_to_md.py`.

### 3.1. Извлечение таблиц по геометрии
- `_PDF_TABLE_STRATEGIES` — **только** стратегия `lines` (стратегия `text`
  убрана). **Проверить:** не теряются ли реальные безбордюрные таблицы с
  фоновой заливкой (должны ловиться по краям прямоугольников). Подобрать
  PDF, где таблица без линий, но с заливкой.
- `_looks_like_table` (фильтр «это таблица»: ≥2 кол., ≥2 многоколоночных
  строк, доля длинных ячеек ≤50%) — нет ли ложных таблиц/потерь реальных.
- `_segment_table_rows`, `_drop_empty_columns`, `_join_continued_tables`
  (склейка переносной таблицы по повтору заголовка) — корректность,
  отсутствие дублей/потерь, защита от пустого списка.
- `_table_to_gfm` — экранирование `|`, схлопывание `\n` в ячейке,
  дополнение рваных строк.

### 3.2. Псевдографика → GFM (`_convert_box_tables` в `tidy`)
- Срабатывает только при `│`/`┃`; не задевает прозу и готовый GFM с
  ASCII-`|`. Слияние двухстрочных шапок, срез внешних рамок.

### 3.3. Чистка PDF-текста (`_clean_pdf_text`)
- `_strip_pdf_furniture`: удаление сквозных колонтитулов (повтор ≥
  `page_count//2`, длина ≤60) и страничных номеров (только если колонтитул
  найден). **Проверить:** не режется ли легитимный повторяющийся
  короткий контент в многостраничном документе; одиночные числа в
  уникальном документе сохраняются.
- `_escape_stray_heading`: экранирование ложных `#`-заголовков **только
  вне** код-фенсов.

### 3.4. Фенсинг кода (`_classify_code_line` / `_fence_code_blocks`)
- Классификатор: русская проза кириллице-доминантна → проза; код —
  латиница-доминантность / команды (`sudo`, CLI `cmd -flag`) / SQL /
  `$VAR` / `{<...>}` / `UUID=`/`deb` / продолжение `\`. Настоящие таблицы
  (`| … |`) и `│` — границы блока.
- **Главные риски (проверить на своём корпусе):**
  - **FP** — проза, обёрнутая в код-блок (наиболее вредно). Цель ≈0.
    Подозрительные паттерны: проза, начинающаяся с пути/`key=val`/латинского
    термина; короткие латинские заголовки.
  - **FN** — пропущенный код (кириллице-доминантные команды без `sudo`/
    CLI-паттерна) — менее вредно, но отметить.
  - Группировка: `neutral`/одиночный пробел не должны рвать блок; gfm/проза
    — рвут.
- **Взаимодействие с `tidy`/`sanitize_markdown` (важно):** фенсинг идёт ДО
  `tidy`, а `sanitize_markdown`/`_convert_box_tables` применяются ко **всему**
  тексту (НЕ fence-aware). Проверено, что `<плейсхолдеры>`, `|`, голые
  ссылки байтово выживают. **Проверить отдельно:** не портит ли sanitize
  контент ВНУТРИ ``` на других входах — автолинки `<scheme://…>`
  (→`<blocked>`), markdown-ссылки `[x](y)`, `javascript:` в код-примере.
  Если портит — оценить, нужна ли fence-aware санитизация.
- `--pdf-tables off` обходит весь PDF-конвейер (фенсинг/чистку/таблицы) и
  уходит на штатный MarkItDown — проверить, что bypass полный.

### 3.5. Зависимости/версия
- `pdfplumber>=0.11.9` и `pypdfium2>=4` объявлены **прямыми** (импорт
  напрямую). Лицензии — MIT/BSD/Apache, без AGPL; cp314-колёса есть.
- Версия `1.3.0` консистентна: `pyproject.toml`, `__version__`, `uv.lock`,
  README, генератор во front-matter, `tomd --version`.

## 4. Точки внимания по проекту в целом

- **Безопасность:** SSRF (per-hop `_check_url_allowed`, `trust_env=False`,
  лимит редиректов, `_is_public_ip` incl. CGNAT), sandbox-worker
  (`subprocess.run` без `shell=True`, strict UTF-8, timeout), очистка
  Markdown (`sanitize_markdown`, порядок `_CTRL_DROP` до санитайза),
  data-URI whitelist, лимиты размера файла/URL.
- **Корректность путей/имён:** развод одинаковых имён (`(2),(3)`),
  `--mirror` без path-traversal, автоопределение кодировки HTML
  (BOM→meta→cyrillic→charset_normalizer; UTF-32 до UTF-16).
- **Поставка:** `uv.lock` в git, `uv lock --check`/`uv sync --frozen` в CI;
  CycloneDX SBOM (runtime+dev), `pip-audit`, license-inventory с
  `--fail-on-forbidden`; Dependabot; CI-матрица Linux 3.10–3.14 + Windows
  3.12/3.14; экшены запинены по commit-SHA.
- **Качество:** 100 pytest-тестов (есть покрытие pdfplumber-glue реальным
  zero-dep PDF), `ruff` чист, строки `convert_to_md.py` ≤79.
- **Документация:** соответствие README/CLAUDE.md фактическому коду и
  флагам; актуальность версии/счётчиков.
- **Установщик `install.ps1`:** идемпотентность, PowerShell-гард, BOM,
  fallback `pip --user`, пункт Send To, кириллица в `.lnk` (см. подводные
  камни в CLAUDE.md). Тестовое покрытие — слабое (1 статический тест);
  оценить риск.
- **Производительность:** 388-страничный PDF конвертируется ~34 c при
  `conversion_timeout` по умолчанию 120 c. Оценить запас на больших файлах
  и медленных машинах; не нужен ли динамический таймаут/прогресс.

## 5. Как воспроизвести (локально, как в CI)

```
uv lock --check && uv sync --frozen
uv run --frozen ruff check convert_to_md.py tests tools
uv run --frozen pytest -q
```

Функциональная проверка PDF (нужны свои PDF — в репозитории бинарных PDF
нет): прогнать `tomd <файл>.pdf -o out -f`, затем оценить во Markdown-вьюере:
- таблицы (геометрия) — структура, отсутствие дублей/слипания;
- код/команды/SQL — внутри ``` , плейсхолдеры `<...>` видны буквально;
- колонтитулы/номера страниц не текут в тело; `#`-комменты не заголовки;
- **FP-чек:** grep строк внутри ``` , которые кириллице-доминантны и без
  маркера кода — целевое значение 0.
Сверить дефолтный путь (sandbox-worker) и `--no-sandbox` — должны давать
идентичный результат.
